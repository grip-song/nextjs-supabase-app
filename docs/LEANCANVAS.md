## 1. 문제 (Problem)

> 소규모 일회성 이벤트를 관리할 때의 불편함

**핵심 문제 3가지**

1. **분산된 이벤트 관리**
   - 생일 파티, 워크샵, 세미나 등 일회성 이벤트 기획 시 카카오톡 단체방 + 메모장 + 계산기 등 여러 툴을 넘나들어야 함
   - 참여 여부 취합에 수동으로 채팅 메시지를 일일이 확인해야 함

2. **참여자 관리의 번거로움**
   - "누가 온다고 했지?" 참여자 명단 추적이 어려움
   - 새 참여자가 생길 때마다 직접 명단을 업데이트해야 함

3. **이벤트 후속 처리 분산**
   - 정산(계좌이체 여부), 카풀 배정 등 이벤트 후속 작업이 채팅방에 분산됨
   - 비용 N분의 1 계산을 별도로 해야 함

**기존 대안과 한계**

| 기존 대안       | 한계                                    |
| --------------- | --------------------------------------- |
| 카카오 단체방   | 메시지에 묻혀 정보 소실, 명단 관리 불가 |
| 카카오 톡캘린더 | 일정 공유만 가능, 참여자 관리 기능 없음 |
| 소모임 앱       | 정기 동호회 중심, 일회성 이벤트 비적합  |
| Eventbrite      | 대규모 유료 이벤트용, 소규모 비적합     |

## 2. 고객 세그먼트 (Customer Segments)

> 5~30명 규모 소규모 이벤트를 자주 만드는 사람들

**주요 고객군**

### A. 이벤트 주최자 (Primary)

- **대상**: 20~40대 직장인, 대학생
- **상황**: 생일 파티, 팀 회식, 동문 모임, 스터디, 워크샵 등을 기획
- **특징**: 카카오톡 링크 공유에 익숙하고, 빠른 온보딩을 선호
- **페인 포인트**: 참여자 명단 취합 + 정산 + 카풀 배정까지 직접 챙겨야 하는 번거로움

### B. 이벤트 참여자 (Secondary)

- **대상**: 초대 링크를 받은 지인
- **상황**: 링크를 클릭해 이벤트 정보 확인 후 참여 신청
- **특징**: 별도 앱 설치 없이 간편하게 참여하고 싶어함

### C. 플랫폼 관리자 (Internal)

- **대상**: Gather 운영팀
- **필요**: 이벤트·사용자 현황 모니터링, 이상 이벤트 삭제

**얼리 어답터**

- 5~30명 규모 행사를 월 1회 이상 기획하는 20~35세 모바일 헤비유저
- 카카오톡으로 소통하는 국내 사용자

## 3. 가치 제안 (Unique Value Proposition)

> **"초대 링크 하나로, 이벤트 생성부터 참여자 관리까지"**

**핵심 메시지**

앱 설치도, 긴 회원가입도 필요 없습니다.  
링크 하나를 카카오톡에 보내면, 참여자들이 Google 계정으로 3초 만에 참여 신청하고,  
주최자는 실시간으로 누가 온다고 했는지 바로 확인할 수 있습니다.

**사용자별 핵심 가치**

| 대상   | 핵심 가치                                                          |
| ------ | ------------------------------------------------------------------ |
| 주최자 | 이벤트 생성 → 초대 링크 공유 → 실시간 참여자 확인까지 3단계로 완료 |
| 참여자 | 링크 클릭 → Google 로그인 → 참여 완료, 별도 앱 설치 불필요         |
| 관리자 | 대시보드에서 전체 현황 한눈에 파악                                 |

## 4. 해결책 (Solution)

> 일회성 이벤트 관리의 3가지 핵심 문제를 해결하는 기능

**MVP 핵심 기능 (Phase 1)**

| #   | 기능                      | 해결하는 문제                            |
| --- | ------------------------- | ---------------------------------------- |
| 1   | **초대 링크 자동 생성**   | 링크 하나로 이벤트 공유 (카카오톡, 문자) |
| 2   | **실시간 참여자 목록**    | Supabase Realtime으로 참여 즉시 업데이트 |
| 3   | **Google OAuth 로그인**   | 3초 만에 가입·참여 완료                  |
| 4   | **이벤트 CRUD**           | 제목·날짜·장소·커버 이미지 관리          |
| 5   | **이벤트 상태 자동 분류** | 예정 / 진행 중 / 종료 자동 분류          |
| 6   | **관리자 대시보드**       | 이벤트·사용자 통합 관리                  |

**Phase 2 예정 기능**

- **정산 관리**: N분의 1 자동 계산 + 송금 완료 체크리스트
- **카풀 관리**: 운전자 등록, 탑승자 신청·배정
- **알림 시스템**: 이벤트 리마인더, 참여자 추가 알림

## 5. 경쟁우위 (Unfair Advantage)

> 쉽게 복사하거나 구매할 수 없는 차별점

**1. 소규모 일회성 이벤트 특화 포지셔닝**

- 5~30명 규모에 완벽히 최적화된 UX (대규모 이벤트 플랫폼과의 명확한 차별화)
- 정기 동호회가 아닌 일회성 이벤트에 집중

**2. 한국 카카오톡 생태계 최적화**

- 초대 링크를 카카오톡으로 공유하는 네이티브 UX
- 국내 사용자 친화적 한국어 인터페이스

**3. 정산·카풀 올인원 통합 (Phase 2)**

- 이벤트 생성부터 정산·카풀 배정까지 하나의 플랫폼
- 이를 통합 제공하는 국내 경쟁 서비스 부재

**4. 완전 무료**

- 광고 없이 핵심 기능을 완전 무료로 제공하여 바이럴 확산 유도

---

## 6. 수익 흐름 (Revenue Stream)

> MVP 단계는 무료 모델로 사용자 확보 우선

**Phase 1 (MVP): 완전 무료**

- 수익 없음, 사용자 확보와 제품 검증 집중
- 무료 Supabase + Vercel 티어로 초기 비용 최소화

**Phase 2 (수익화 단계)**

| 수익 모델         | 내용                                          | 예상 시점           |
| ----------------- | --------------------------------------------- | ------------------- |
| **프리미엄 구독** | 월 구독료로 고급 기능 제공 (정산, 카풀, 알림) | MAU 1,000명 달성 후 |
| **기업 솔루션**   | 팀 빌딩, 기업 세미나 전용 요금제              | Phase 2 이후        |
| **광고 (선택)**   | 이벤트 관련 제휴 광고 (장소, 케이터링 등)     | Phase 3 이후        |

## 7. 비용 구조 (Cost Structure)

> 초기 비용을 최소화하는 서버리스 BaaS 아키텍처

**고정 비용**

| 항목     | 초기 (무료 티어)             | 성장 단계          |
| -------- | ---------------------------- | ------------------ |
| Supabase | 무료 (500MB DB, 1GB Storage) | $25/월~ (Pro 플랜) |
| Vercel   | 무료 (Hobby 플랜)            | $20/월~ (Pro 플랜) |
| 도메인   | ~$15/년                      | 동일               |

**변동 비용**

- Supabase Storage: 이미지 업로드 용량 기준 ($0.021/GB)
- Supabase Realtime: 동시 접속 증가 시 Pro 플랜 필요
- 개발 인력: 솔로 개발자 (현재 인건비 없음)

**MVP 예상 월 운영 비용**: $0 ~ $35 (트래픽에 따라 변동)

## 8. 주요 지표 (Key Metrics)

> 제품이 성공하고 있는지 판단하는 핵심 수치

**1차 목표 (MVP 출시 후 1개월)**

| 지표                    | 목표값      | 측정 방법                     |
| ----------------------- | ----------- | ----------------------------- |
| 주간 생성 이벤트 수     | **50개/주** | events 테이블 COUNT           |
| 월간 활성 사용자 (MAU)  | **200명**   | Supabase Auth unique users    |
| 초대 링크 → 참여 전환율 | **70%**     | event_participants JOIN users |
| 이벤트당 평균 참여자 수 | **8명**     | AVG(participant COUNT)        |
| 7일 내 재방문율         | **40%**     | returning users 분석          |

**2차 목표 (Phase 2 출시 후 3개월)**

| 지표                | 목표값   |
| ------------------- | -------- |
| 주간 생성 이벤트 수 | 200개/주 |
| 월간 활성 사용자    | 1,000명  |
| 정산 기능 사용률    | 60%      |
| 카풀 기능 사용률    | 40%      |

**선행 지표 (Leading Indicators)**

- 초대 링크 클릭 → 이벤트 미리보기 조회 수
- 이벤트 미리보기 → Google 로그인 시도율
- 주최자의 두 번째 이벤트 생성율 (리텐션 핵심)

## 9. 채널 (Channels)

> 고객에게 도달하고 가치를 전달하는 경로

**획득 채널 (Acquisition)**

| 채널                | 방식                                           | 우선순위      |
| ------------------- | ---------------------------------------------- | ------------- |
| **카카오톡 바이럴** | 초대 링크 공유를 통한 자연 확산                | ⭐⭐⭐ 최우선 |
| **입소문 (WoM)**    | 이벤트 참여 경험 후 주최자로 전환              | ⭐⭐⭐ 최우선 |
| **SEO**             | "생일파티 참여자 모집", "이벤트 초대장" 키워드 | ⭐⭐ 중기     |
| **소셜 미디어**     | 인스타그램·트위터 이벤트 주최 후기 공유        | ⭐ 장기       |

**활성화 채널 (Activation)**

- 초대 링크 클릭 → Google 로그인 → 자동 참여 완료 (3단계 온보딩)
- 이벤트 상세 페이지의 "참여 완료!" 확인 메시지

**리텐션 채널 (Retention)**

- 실시간 참여자 목록 업데이트 (참여의 재미 제공)
- 이벤트 상태 자동 분류로 지속적인 앱 방문 유도
- (Phase 2) 이벤트 리마인더 이메일·푸시 알림
